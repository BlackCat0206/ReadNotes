# 189_轮转数组

---- Lee

## 一、题目

给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

 

**示例 1:**

```
输入: nums = [1,2,3,4,5,6,7], k = 3
输出: [5,6,7,1,2,3,4]
解释:
向右轮转 1 步: [7,1,2,3,4,5,6]
向右轮转 2 步: [6,7,1,2,3,4,5]
向右轮转 3 步: [5,6,7,1,2,3,4]
```

**示例 2:**

```
输入：nums = [-1,-100,3,99], k = 2
输出：[3,99,-1,-100]
解释: 
向右轮转 1 步: [99,-1,-100,3]
向右轮转 2 步: [3,99,-1,-100]
```

**提示：**

- `1 <= nums.length <= 105`
- `-231 <= nums[i] <= 231 - 1`
- `0 <= k <= 105`

**进阶：**

- 尽可能想出更多的解决方案，至少有 **三种** 不同的方法可以解决这个问题。
- 你可以使用空间复杂度为 `O(1)` 的 **原地** 算法解决这个问题吗？

## 二、思路

* 逐步位移替换：每次记录末尾元素，数组位移，将数组首位填充记录的末尾元素。（对于巨型数组，会超时！！！）

* 反转：感谢灵神ಥ_ಥ，[题解链接](https://leetcode.cn/problems/rotate-array/solutions/2784427/tu-jie-yuan-di-zuo-fa-yi-tu-miao-dong-py-ryfv/)

  * **注意旋转边界**：若k  > intervals；需要旋转的元素是 `k %= intervals.size();`
  * 把 [1,2,3,4,5,6,7] 看成 [5,6,7]+[1,2,3,4]，首先要保证 [5,6,7] 在 [1,2,3,4] 前面，这可以通过反转 [1,2,3,4,5,6,7] 得到。
  * 反转后数组变成 [7,6,5,4,3,2,1]，拆分成 [7,6,5]+[4,3,2,1]，对比最终目标，只需把 [7,6,5] 反转，把 [4,3,2,1] 反转，就得到了 [5,6,7,1,2,3,4]。

  ![lc189.png](https://pic.leetcode.cn/1716207476-TyuIlk-lc189.png)

  这用到了「负负得正」的想法：**把一个子数组反转两次，子数组的元素顺序不变**。

## 三、代码

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        // 入参校验
        if (nums.empty() || k <= 0){
            return;
        }

        // 若旋转的长度超过数组长度
        k %= nums.size();

        reverse(nums.begin(), nums.end());
        reverse(nums.begin(), nums.begin() + k);
        reverse(nums.begin() + k, nums.end());
        return;
    }
};
```
