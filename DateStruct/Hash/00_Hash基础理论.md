## 一、概述

本章为Hash算法相关理论支持，要求记住容器`set`、`mulitset`、`unordered_set`、`map`、`multimap`、`unoredered_map`底层实现、值数值是否可以修改、能否更改、查询效率、增减效率。

哈希表(Hash table)的本质是一种映射，就像数组就是最简单的Hash函数，其将下标和数值进行了隐射。

## 二、重点概念

### 2.1 `set`容器

| 集合                | 底层实现 | 是否有序 | 数值是否可以重复 | 数值是否可以更改 | 查询效率  | 增删效率 | 适用场景                                                     |
| ------------------- | -------- | -------- | ---------------- | ---------------- | --------- | -------- | ------------------------------------------------------------ |
| `std::set`          | 红黑树   | 有序     | 否               | 否               | O(log n)  | O(log n) | 有序+去重（存储有序的唯一 ID 列表（如用户 ID、订单号））     |
| `std::multiset`     | 红黑树   | 有序     | 是               | 否               | O (log n) | O(log n) | 有序+允许重复（日程安排（存储多个时间点的事件，允许同一时间有多个事件，按时间排序）） |
| `std::oredered_set` | 哈希表   | 无序     | 否               | 否               | O(1)      | O(1)     | 去重+急速查找（缓存去重（如判断 URL 是否已访问过，仅需精准查询，无需排序）；） |

### 2.2 `map`容器

| 集合            | 底层实现 | 是否有序 | 数值是否可以重复 | 数值是否可以更改 | 查询效率 | 增删效率 | 适用场景                                                     |
| --------------- | -------- | -------- | ---------------- | ---------------- | -------- | -------- | ------------------------------------------------------------ |
| `map`           | 红黑树   | 有序     | key不可重复      | key不可以修改    | O(log n) | O(log n) | 有序且一个键值对应一个数值（键 = 时间戳（唯一），值 = 日志内容，需查找某个时间段内的日志；） |
| `multimap`      | 红黑树   | 有序     | key可以重复      | key不可以修改    | O(log n) | O(log n) | 一个键对于多个重复值（姓氏）                                 |
| `unordered_map` | 哈希表   | 无序     | key不可重复      | key不可以修改    | O(1)     | O(1)     | 无序且要求急速查找。                                         |
