## 二、内容

### 2.1 代码组织结构与分离编译

大型程序的核心需求是模块化，C++ 通过**分离编译（separate compilation）** 实现代码拆分，将类的声明、函数原型放在头文件（`.h/.hpp`），实现放在源文件（`.cpp`），核心原则是 “声明与定义分离”。

#### 2.1.1 头文件的设计原则

- 头文件应包含**声明性内容**：类声明、函数原型、`extern`变量声明、模板声明 / 定义、`inline`函数定义（需在所有使用处可见）；
- 避免包含**定义性内容**：普通函数 / 变量定义（除非`inline`/`static`/`constexpr`），防止重复定义错误；
- 头文件保护：使用`#ifndef`/`#define`/`#endif`或`#pragma once`避免重复包含。

```cpp
// Sales_data.h - 头文件保护
#ifndef SALES_DATA_H
#define SALES_DATA_H

#include <string>
// 类声明（定义）
class Sales_data {
public:
    std::string isbn() const; // 成员函数声明
private:
    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;
};
// inline函数定义（必须在头文件）
inline double avg_price(const Sales_data& sd) {
    return sd.revenue / sd.units_sold;
}

#endif // SALES_DATA_H
```

```cpp
// Sales_data.cpp - 源文件
#include "Sales_data.h"
// 成员函数实现
std::string Sales_data::isbn() const {
    return bookNo;
}
```

#### 2.1.2 编译与链接流程

- 编译阶段：每个源文件独立编译为目标文件（`.o/.obj`），编译器仅检查语法、类型匹配，不检查函数 / 类的实现是否存在；
- 链接阶段：链接器将所有目标文件、标准库合并为可执行文件，解析跨文件的函数 / 类引用，未找到实现则报 “未定义引用” 错误。

### 2.2 命名空间（namespace）

命名空间用于解决大型程序的**名字冲突**问题，将全局作用域划分为多个独立的命名空间作用域，隔离不同模块的标识符。

#### 2.2.1 命名空间的定义

```cpp
// 命名空间定义（可分散在多个文件）
namespace cppprimer {
    class Sales_data { /* ... */ };
    double calc_tax(const Sales_data&);
    namespace detail { // 嵌套命名空间
        double helper_func(); // 内部辅助函数
    }
}
```

- 全局命名空间：默认作用域，未显式声明的标识符属于此；
- 匿名命名空间：`namespace { /* ... */ }`，仅在当前文件可见，替代`static`修饰全局变量 / 函数。

#### 2.2.2 命名空间的使用

##### （1）作用域解析符（::）

直接指定命名空间，无歧义访问标识符：

```cpp
cppprimer::Sales_data sd;
double tax = cppprimer::calc_tax(sd);
double h = cppprimer::detail::helper_func();
```

##### （2）using 声明（using declaration）

引入单个标识符到当前作用域：

```cpp
using cppprimer::Sales_data;
Sales_data sd; // 等价于cppprimer::Sales_data
```

##### （3）using 指示（using directive）

引入整个命名空间的所有标识符（慎用，可能重新引入名字冲突）：

```cpp
using namespace cppprimer;
Sales_data sd;
calc_tax(sd);
```

#### 2.2.3 命名空间的别名

简化长命名空间名称：

```cpp
namespace cp = cppprimer;
cp::Sales_data sd;
```

### 2.3 多重继承与虚继承

#### 2.3.1 多重继承

一个类继承多个基类，语法：`class 派生类 : 访问说明符 基类1, 访问说明符 基类2 { ... }`。

```cpp
class Base1 {
public:
    void func1();
};
class Base2 {
public:
    void func2();
};
// 多重继承Base1和Base2
class Derived : public Base1, public Base2 {
public:
    void func3();
};

Derived d;
d.func1(); // 调用Base1::func1
d.func2(); // 调用Base2::func2
```

- 二义性问题：若多个基类有同名成员，需显式指定基类作用域：

```cpp
class Base1 { public: int val; };
class Base2 { public: int val; };
class Derived : public Base1, public Base2 {};

Derived d;
d.Base1::val = 10; // 显式指定，避免二义性
```

#### 2.3.2 虚继承

解决多重继承中的 “菱形继承”（派生类间接多次继承同一基类）导致的基类成员冗余问题，使共享的基类在派生类中只保留一份副本。

- 语法：继承时加`virtual`关键字；
- 虚基类的初始化：由最终派生类负责初始化虚基类（而非直接派生类）。

```cpp
// 虚基类
class Base {
public:
    Base(int v) : val(v) {}
    int val;
};
// 虚继承Base
class Derived1 : virtual public Base {
public:
    Derived1() : Base(0) {} // 最终派生类未初始化时，此处仅为占位
};
// 虚继承Base
class Derived2 : virtual public Base {
public:
    Derived2() : Base(0) {}
};
// 最终派生类，负责初始化虚基类Base
class FinalDerived : public Derived1, public Derived2 {
public:
    FinalDerived() : Base(10) {} // 显式初始化虚基类
};

FinalDerived fd;
fd.val = 20; // 仅一份val，无冗余
```

### 2.4 运行时类型识别（RTTI）

RTTI（Run-Time Type Identification）提供程序运行时获取对象类型信息的能力，核心组件：`dynamic_cast`、`typeid`。

#### 2.4.1 dynamic_cast

将基类指针 / 引用安全转换为派生类指针 / 引用，仅当对象实际类型匹配时转换成功，否则：

- 指针转换：返回`nullptr`；
- 引用转换：抛出`bad_cast`异常。

```cpp
class Base { virtual void func() {} }; // 必须有虚函数，RTTI才生效
class Derived : public Base { void func() override {} };

Base* bp = new Derived;
// 向下转换：Base* → Derived*
Derived* dp = dynamic_cast<Derived*>(bp);
if (dp) { // 转换成功，dp非空
    dp->func();
}

Base& br = *bp;
try {
    Derived& dr = dynamic_cast<Derived&>(br);
} catch (std::bad_cast& e) {
    std::cerr << "转换失败：" << e.what() << std::endl;
}
```

#### 2.4.2 typeid

返回`type_info`对象，标识对象的实际类型：

```cpp
#include <typeinfo>

Base* bp1 = new Base;
Base* bp2 = new Derived;

// 比较类型
if (typeid(*bp1) == typeid(Base)) {
    std::cout << "bp1指向Base对象" << std::endl;
}
if (typeid(*bp2) == typeid(Derived)) {
    std::cout << "bp2指向Derived对象" << std::endl;
}
// 获取类型名
std::cout << typeid(*bp2).name() << std::endl; // 输出Derived（具体名称依赖编译器）
```

- 注意：`typeid`作用于指针时，返回指针类型（而非指向对象的类型），需解引用指针。

### 2.5 异常处理的高级用法

#### 2.5.1 异常规范（C++11 弃用 noexcept）

C++98 的`throw()`异常规范（指定函数可抛出的异常类型）已被废弃，C++11 引入`noexcept`，表明函数不会抛出任何异常：

```cpp
// 声明函数不会抛出异常
void func() noexcept;
// 条件性noexcept：表达式为true时，函数不抛异常
template <typename T>
void swap(T& a, T& b) noexcept(std::is_nothrow_swappable<T>::value);
```

- `noexcept`的作用：编译器可优化代码，若`noexcept`函数抛出异常，程序直接终止（调用`std::terminate`）。

#### 2.5.2 异常类的继承

自定义异常类通常继承标准异常类（`<stdexcept>`），实现多态异常处理：

```cpp
#include <stdexcept>
#include <string>

// 自定义异常类
class SalesException : public std::runtime_error {
public:
    // 调用基类构造函数
    SalesException(const std::string& msg) : std::runtime_error(msg) {}
};

// 抛出自定义异常
void calc_sales() {
    if (/* 错误条件 */) {
        throw SalesException("销售额计算错误：数据为空");
    }
}

// 捕获异常（多态）
try {
    calc_sales();
} catch (const std::runtime_error& e) { // 捕获所有runtime_error派生类
    std::cerr << "运行时错误：" << e.what() << std::endl;
}
```

#### 2.5.3 异常的栈展开

抛出异常时，编译器会销毁当前函数栈中的局部对象（包括类类型对象，调用析构函数），直到找到匹配的`catch`子句，此过程为 “栈展开”；若未找到`catch`，调用`std::terminate`终止程序。

- 析构函数应保证不抛出异常（否则栈展开时可能导致程序崩溃）。

### 2.6 动态内存管理扩展

#### 2.6.1 智能指针的高级用法

##### （1）自定义删除器

`shared_ptr`/`unique_ptr`可指定自定义删除器，处理非常规资源（如文件句柄、网络连接）：

```cpp
#include <memory>
#include <fstream>

// shared_ptr自定义删除器（函数）
void close_file(std::ofstream* fp) {
    fp->close();
    delete fp;
}
// 创建shared_ptr，绑定删除器
std::shared_ptr<std::ofstream> fp(new std::ofstream("test.txt"), close_file);

// unique_ptr自定义删除器（lambda）
auto del = [](int* p) {
    std::cout << "删除int对象" << std::endl;
    delete p;
};
std::unique_ptr<int, decltype(del)> up(new int(10), del);
```

##### （2）weak_ptr

解决`shared_ptr`循环引用问题，是 “弱引用”，不增加引用计数，可判断`shared_ptr`指向的对象是否存在：

```cpp
#include <memory>

class Node {
public:
    std::weak_ptr<Node> next; // 弱引用，避免循环引用
    ~Node() { std::cout << "Node析构" << std::endl; }
};

auto n1 = std::make_shared<Node>();
auto n2 = std::make_shared<Node>();
n1->next = n2;
n2->next = n1;
// 无循环引用，n1/n2引用计数为1，离开作用域后析构
```

#### 2.6.2 动态数组

`unique_ptr`支持管理动态数组，`shared_ptr`需自定义删除器：

```cpp
// unique_ptr管理动态数组
std::unique_ptr<int[]> up(new int[5]{1,2,3,4,5});
up[0] = 10; // 支持[]访问
up.reset(); // 自动调用delete[]

// shared_ptr管理动态数组（需指定删除器）
std::shared_ptr<int> sp(new int[5], [](int* p) { delete[] p; });
// 通过get()访问数组
*(sp.get() + 1) = 20;
```

### 2.7 预处理指令高级用法

#### 2.7.1 条件编译

根据宏定义决定编译哪些代码，常用于跨平台开发、调试模式：

```cpp
// 调试模式
#define DEBUG

#ifdef DEBUG
#define LOG(msg) std::cout << "[DEBUG] " << msg << std::endl;
#else
#define LOG(msg)
#endif

void func() {
    LOG("进入func函数"); // DEBUG定义时，输出日志；否则无操作
    // 业务逻辑
}

// 跨平台编译
#ifdef _WIN32
// Windows平台代码
#else
// Linux/Mac平台代码
#endif
```

#### 2.7.2 宏的高级用法

- 带参数宏：注意括号，避免运算符优先级问题；
- `#`（字符串化）、`##`（连接符）：

```cpp
// 带参数宏（加括号避免优先级问题）
#define ADD(a, b) ((a) + (b))
int res = ADD(1+2, 3*4); // ((1+2)+(3*4)) = 15

// 字符串化
#define STR(x) #x
std::cout << STR(123) << std::endl; // 输出"123"

// 连接符
#define CONCAT(a, b) a##b
int num123 = 100;
std::cout << CONCAT(num, 123) << std::endl; // 输出100
```

## 三、小结

第 18 章聚焦大型 C++ 程序的开发范式，核心围绕 “模块化、可维护性、可扩展性” 展开：

1. 分离编译与头文件设计是模块化的基础，保证代码拆分后的编译与链接正确性；
2. 命名空间解决多模块名字冲突，是大型项目的必备工具；
3. 多重继承与虚继承拓展了类的复用能力，需注意二义性和菱形继承问题；
4. RTTI 提供运行时类型信息，支持动态类型安全转换，但应慎用（可能降低性能）；
5. 异常处理的高级用法（`noexcept`、自定义异常类）提升了程序的错误处理能力；
6. 动态内存管理扩展（智能指针自定义删除器、`weak_ptr`）解决了非常规资源管理和循环引用问题；
7. 预处理指令的条件编译、宏技巧支撑跨平台和调试需求。

大型程序开发的核心是平衡 “复用性、可读性、性能”，需合理运用本章的技术，遵循 “高内聚、低耦合” 的设计原则，同时兼顾代码的可移植性和健壮性。
