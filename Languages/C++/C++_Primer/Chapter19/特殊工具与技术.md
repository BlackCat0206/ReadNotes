## 二、内容

### 2.1 类型转换运算符

C++ 提供了四种显式的类型转换运算符（命名的强制类型转换），用于替代旧式的强制类型转换，使转换意图更清晰，也更易于排查问题。

#### 2.1.1 `static_cast`

- **适用场景**：用于编译器可自动执行的类型转换（隐式转换）的显式版本，如算术类型转换、非`const`转`const`、子类指针 / 引用转父类（向上转型）、void 指针转其他类型指针等。
- **特点**：编译期完成转换，不进行运行时类型检查，安全性依赖于程序员的判断。
- **示例**：

```cpp
// 算术类型转换
double num = 3.14;
int i = static_cast<int>(num); // 3

// 类类型向上转型
class Base {};
class Derived : public Base {};
Derived d;
Base* b = static_cast<Base*>(&d);
```

#### 2.1.2 `const_cast`

- **适用场景**：唯一能修改表达式`const`/`volatile`属性的转换符，仅改变常量性，不改变类型本身。
- **注意事项**：若修改原本声明为`const`的对象，行为未定义；仅适用于指针、引用或指向对象成员的指针。
- **示例**：

```cpp
const std::string str = "hello";
std::string& non_const_str = const_cast<std::string&>(str);
non_const_str = "world"; // 若str是const对象，此操作未定义

// 合法场景：解除函数参数的const限制（参数本身非const）
void func(const int& x) {
    int& nx = const_cast<int&>(x);
    nx = 10;
}
int a = 5;
func(a); // 合法，a本身非const
```

#### 2.1.3 `reinterpret_cast`

- **适用场景**：低级别的类型转换，将一种类型的指针 / 引用转换为另一种无关类型的指针 / 引用，或指针与整数类型的转换。
- **特点**：依赖于编译器和系统架构，高度不可移植，仅用于底层编程（如硬件交互、内存操作）。
- **示例**：

```cpp
int i = 0x12345678;
char* p = reinterpret_cast<char*>(&i);
// 按字节访问i的内存，结果依赖于系统字节序
```

#### 2.1.4 `dynamic_cast`

- **适用场景**：用于运行时类型识别（RTTI），仅适用于指针或引用，且只能用于包含虚函数的类层次结构（保证多态性）。
- **核心功能**：安全地将基类指针 / 引用转换为派生类指针 / 引用（向下转型），或跨类层次的转换。
- **返回值规则**：
  - 指针版：转换失败返回`nullptr`；
  - 引用版：转换失败抛出`std::bad_cast`异常。
- **示例**：

```cpp
class Base { virtual void f() {} };
class Derived : public Base { void f() override {} };

Base* b = new Derived;
// 向下转型：基类指针转派生类指针
Derived* d = dynamic_cast<Derived*>(b); // 成功，d指向Derived对象

Base* b2 = new Base;
Derived* d2 = dynamic_cast<Derived*>(b2); // 失败，d2为nullptr
```

### 2.2 运行时类型识别（RTTI）

RTTI（Run-Time Type Identification）提供了在运行时确定对象类型的能力，主要依赖`dynamic_cast`和`typeid`运算符。

#### 2.2.1 `typeid`运算符

- **作用**：返回`std::type_info`对象，描述表达式的类型信息（类型名、是否相等）。
- **使用规则**：
  - 作用于对象时，若对象是多态类型（含虚函数），返回实际类型；否则返回静态类型；
  - 作用于类型名时，直接返回该类型的`type_info`；
  - 比较两个`typeid`结果可判断类型是否相同。
- **示例**：

```cpp
#include <typeinfo>

Base* b = new Derived;
// 多态类型，返回Derived类型
std::cout << typeid(*b).name() << std::endl; 
// 静态类型，返回Base类型
std::cout << typeid(b).name() << std::endl;  

if (typeid(*b) == typeid(Derived)) {
    std::cout << "b指向Derived对象" << std::endl;
}
```

#### 2.2.2 RTTI 的注意事项

- 仅对多态类型（含虚函数）能正确返回动态类型；
- 过度使用 RTTI 通常意味着设计缺陷（应优先使用多态）；
- 性能开销：`dynamic_cast`和`typeid`会增加运行时开销，需谨慎使用。

### 2.3 枚举类型

枚举类型（enumeration）用于定义一组命名的整数常量，分为**不限定作用域枚举**（传统枚举）和**限定作用域枚举**（C++11）。

#### 2.3.1 限定作用域枚举（`enum class`/`enum struct`）

- **特性**：

  - 枚举成员作用域限定在枚举类型内，避免名字冲突；
  - 不能隐式转换为算术类型，需显式转换；
  - 可指定底层类型（默认`int`）。

  

- **示例**：

```cpp
enum class Color { Red, Green, Blue };
Color c = Color::Red; // 必须加作用域
// 显式转换为int
int i = static_cast<int>(Color::Green); // 1

// 指定底层类型
enum class Status : unsigned char { Ok = 0, Error = 1 };
```

#### 2.3.2 不限定作用域枚举

- **特性**：
  - 枚举成员作用域与枚举类型相同，易引发名字冲突；
  - 可隐式转换为算术类型；
  - 底层类型由编译器决定（通常为`int`）。
- **示例**：

```cpp
enum Direction { Left, Right, Up, Down };
Direction d = Left;
int j = Right; // 隐式转换，j=1

// 指定底层类型（C++11）
enum Score : short { A = 90, B = 80, C = 70 };
```

#### 2.3.3 枚举成员的初始化

- 枚举成员默认从 0 开始递增，可显式指定值；
- 允许重复值，但通常不推荐。

```cpp
enum ErrorCode {
    Success = 0,
    FileNotFound = 1,
    NetworkError = 1, // 允许重复
    PermissionDenied = 2
};
```

### 2.4 类成员指针

成员指针指向类的成员（数据成员或成员函数），而非对象的成员，需结合对象 / 指针才能访问。

#### 2.4.1 数据成员指针

- **声明语法**：`类型 类名::*指针名`；
- **使用方式**：通过`.*`（对象）或`->*`（指针）访问。
- **示例**：

```cpp
class Sales_data {
public:
    std::string bookNo;
    unsigned units_sold = 0;
};

// 声明指向Sales_data的bookNo成员的指针
std::string Sales_data::*p_book = &Sales_data::bookNo;
Sales_data sd{"978-0134400315", 10};
// 访问sd的bookNo成员
std::cout << sd.*p_book << std::endl; 

Sales_data* p_sd = &sd;
// 访问p_sd指向对象的bookNo成员
std::cout << p_sd->*p_book << std::endl;
```

#### 2.4.2 成员函数指针

- **声明语法**：`返回类型 (类名::*指针名)(参数列表) const(可选)`；
- **特点**：需匹配函数的参数列表、返回值、`const`属性；
- **示例**：

```cpp
class Sales_data {
public:
    std::string isbn() const { return bookNo; }
    double avg_price() const { 
        return units_sold ? revenue/units_sold : 0; 
    }
private:
    std::string bookNo;
    unsigned units_sold = 0;
    double revenue = 0.0;
};

// 声明指向isbn成员函数的指针
std::string (Sales_data::*p_isbn)() const = &Sales_data::isbn;
Sales_data sd{"978-0134400315", 10, 99.9};
// 调用成员函数
std::cout << (sd.*p_isbn)() << std::endl; 

Sales_data* p_sd = &sd;
std::cout << (p_sd->*p_isbn)() << std::endl;
```

### 2.5 嵌套类

嵌套类（nested class）是定义在另一个类内部的类，用于将辅助类封装在外部类的作用域内，减少全局命名空间污染。

#### 2.5.1 嵌套类的特性

- 嵌套类的作用域包含在外部类中，需通过`外部类::嵌套类`访问；
- 嵌套类与外部类相互独立，嵌套类不能直接访问外部类的成员（需通过外部类对象 / 指针）；
- 外部类可访问嵌套类的所有成员（无论`public`/`private`）；
- 嵌套类的成员函数可定义在外部类外，需指定完整作用域。

#### 2.5.2 示例

```cpp
class Screen {
public:
    // 嵌套类：表示屏幕上的位置
    class Cursor {
    public:
        Cursor(int x, int y) : pos_x(x), pos_y(y) {}
        void move(int dx, int dy) {
            pos_x += dx;
            pos_y += dy;
        }
    private:
        int pos_x, pos_y;
    };

    Screen(int w, int h) : width(w), height(h), cursor(0, 0) {}
    void move_cursor(int dx, int dy) {
        cursor.move(dx, dy); // 访问嵌套类对象
    }
private:
    int width, height;
    Cursor cursor; // 嵌套类作为成员
};

// 外部定义嵌套类的成员函数（若需要）
void Screen::Cursor::move(int dx, int dy) {
    pos_x += dx * 2;
    pos_y += dy * 2;
}

// 使用嵌套类
Screen::Cursor c(5, 5);
c.move(1, 1);
```

### 2.6 局部类

局部类（local class）是定义在函数内部的类，仅在该函数作用域内有效。

#### 2.6.1 局部类的特性

- 作用域仅限于包含它的函数，不能被外部访问；
- 局部类的成员函数必须定义在类内部（不能在外部定义）；
- 不能声明静态成员（静态成员需在全局作用域分配存储，局部类无此权限）；
- 可访问外部函数的`static`变量和外部作用域的类型名，但不能访问非静态局部变量（需通过参数传递）；
- 局部类可继承外部类，也可被嵌套在其他局部类中。

#### 2.6.2 示例

```cpp
void func() {
    static int global_val = 10; // 静态局部变量
    int local_val = 20; // 非静态局部变量

    // 局部类
    class LocalClass {
    public:
        LocalClass(int x) : val(x) {}
        void print() {
            std::cout << "global_val: " << global_val << std::endl; // 可访问static变量
            // std::cout << local_val << std::endl; // 错误：不能访问非静态局部变量
            std::cout << "val: " << val << std::endl;
        }
    private:
        int val;
    };

    LocalClass lc(30);
    lc.print();
}

// LocalClass lc; // 错误：局部类仅在func内可见
```

### 2.7 固有的不可移植特性

C++ 标准允许编译器实现一些依赖于具体平台的特性，这些特性不可移植，仅用于特定场景。

#### 2.7.1 位域（bit-field）

- **用途**：将整数类型的成员压缩到指定位数的存储空间，节省内存（如硬件寄存器、网络协议解析）；
- **声明语法**：`类型 成员名 : 位数;`；
- **注意事项**：位数不能超过所属类型的大小，跨平台时字节序、对齐规则可能不同。
- **示例**：

```cpp
// 存储状态标志（仅需3位）
struct Flags {
    unsigned int read : 1;   // 1位：读权限
    unsigned int write : 1;  // 1位：写权限
    unsigned int execute : 1;// 1位：执行权限
    unsigned int reserved : 29; // 预留29位
};

Flags f;
f.read = 1;
f.write = 0;
std::cout << sizeof(f) << std::endl; // 4（unsigned int大小）
```

#### 2.7.2 `volatile`限定符

- **作用**：告诉编译器该对象可能被程序外的因素（如硬件、其他线程）修改，禁止编译器对其进行优化（如缓存值到寄存器）；
- **适用场景**：硬件寄存器访问、多线程共享变量（需配合原子操作）；
- **示例**：

```cpp
volatile int reg = 0; // 表示硬件寄存器，值可能被外部修改
reg = 1; // 直接写入内存，不缓存到寄存器
int val = reg; // 直接读取内存，不使用寄存器缓存值
```

#### 2.7.3 链接指示符（`extern "C"`）

- **用途**：指定函数 / 变量使用 C 语言的链接规则，用于 C++ 程序调用 C 语言库；
- **特性**：C 语言无函数重载，`extern "C"`修饰的函数名不会被编译器重整（mangling）；
- **示例**：

```cpp
// 声明C语言函数
extern "C" {
    void c_func(int); // C语言实现的函数
}

// 定义可被C语言调用的C++函数
extern "C" void cpp_func() {
    std::cout << "Called from C" << std::endl;
}
```

## 三、小结

第 19 章聚焦 C++ 的高级工具与技术，补充了核心语法之外的实用特性：类型转换运算符明确了不同场景下的类型转换规则，RTTI 支持运行时类型判断，枚举类型优化了常量定义，成员指针实现了对类成员的间接访问，嵌套类和局部类提升了代码封装性，而位域、`volatile`、`extern "C"`等特性则满足了底层编程、跨语言调用等特殊需求。这些技术虽不常用，但在高性能编程、系统开发、跨平台适配等场景中不可或缺，使用时需兼顾语法规则与平台特性，平衡灵活性与可移植性。
