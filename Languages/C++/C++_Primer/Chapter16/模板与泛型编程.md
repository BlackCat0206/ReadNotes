## 一、概述

本篇为《C++ Primer》（第 5 版）第 16 章读书笔记，主要内容为模板与泛型编程相关基本概述、特性、使用方法及核心原理。泛型编程的核心思想是**编写与类型无关的代码**，模板是实现泛型编程的基础，允许我们定义可复用的函数和类，适配不同的数据类型，大幅提升代码的灵活性和复用性。

## 二、内容

模板与普通函数 / 类的核心区别：

- 普通函数 / 类针对特定类型编写，复用性受限；
- 模板是 “代码蓝图”，编译时根据传入的类型参数生成具体的函数 / 类（实例化），实现一次编写、多类型适配。

C++ 模板主要分类：

| 模板类型     | 特点                                                         |
| :----------- | :----------------------------------------------------------- |
| 函数模板     | 定义通用的函数，支持不同类型的参数和返回值，自动推导或显式指定类型参数 |
| 类模板       | 定义通用的类，类的成员变量、成员函数可依赖类型参数，需显式指定类型实例化 |
| 模板参数     | 分为类型参数（typename/class）和非类型参数（常量表达式），支持默认参数 |
| 模板特化     | 对特定类型定制模板的实现，分为全特化和偏特化（部分特化）     |
| 可变参数模板 | 接受数量可变的模板参数，适用于参数个数不确定的场景（如 std::tuple） |

### 2.1 函数模板

#### 2.1.1 基本定义与使用

函数模板以`template`关键字开头，后跟模板参数列表（尖括号包裹），参数列表中用`typename`或`class`声明类型参数（二者等价）。

```cpp
// 函数模板：交换两个任意类型的值
template <typename T>
void swap(T& a, T& b) {
    T temp = a;
    a = b;
    b = temp;
}

// 使用：自动推导类型
int x = 1, y = 2;
swap(x, y); // 推导T为int，实例化swap<int>

double m = 3.14, n = 6.28;
swap(m, n); // 推导T为double，实例化swap<double>

// 显式指定类型（避免推导歧义）
swap<int>(x, y); // 强制使用int类型
```

#### 2.1.2 模板参数与函数参数

- 模板参数：可包含多个类型参数，也可包含非类型参数（必须是常量表达式，如整数、指针、引用）；
- 函数参数：模板函数的参数类型可依赖模板参数，支持默认参数。

```cpp
// 多类型参数的函数模板
template <typename T, typename U>
auto add(const T& a, const U& b) -> decltype(a + b) {
    return a + b;
}

int i = 10;
double d = 20.5;
auto res = add(i, d); // T=int，U=double，res=30.5（double类型）

// 非类型模板参数（编译期常量）
template <typename T, size_t N>
T array_sum(const T (&arr)[N]) { // N是数组长度，编译期确定
    T sum = 0;
    for (size_t i = 0; i < N; ++i) {
        sum += arr[i];
    }
    return sum;
}

int arr[] = {1, 2, 3, 4};
int sum = array_sum(arr); // N=4，推导T=int，sum=10
```

#### 2.1.3 模板实参推导

编译器根据函数实参的类型推导模板参数的类型，推导规则：

- 忽略顶层 const：实参是`const int`，推导 T 为`int`；
- 引用 / 指针的推导：保留底层 const，实参是`const int&`，推导 T 为`const int`；
- 数组 / 函数退化为指针：实参是数组`int arr[5]`，推导 T 为`int*`（除非参数是数组引用）。

```cpp
template <typename T>
void func(T param);

const int ci = 5;
func(ci); // T=int（忽略顶层const）

const int& ri = ci;
func(ri); // T=int（忽略顶层const，ri是引用，推导为int）

template <typename T>
void func2(T& param);

func2(ci); // T=const int（保留底层const，param是const int&）

int arr[10];
func(arr); // T=int*（数组退化为指针）
func2(arr); // T=int[10]（参数是引用，保留数组类型）
```

#### 2.1.4 函数模板重载

函数模板可与普通函数重载，也可与其他模板重载，匹配规则：

1. 优先匹配普通函数；
2. 若普通函数不匹配，匹配最特化的模板；
3. 若多个模板匹配度相同，编译报错（二义性）。

```cpp
// 普通函数
void print(int x) {
    std::cout << "普通函数：" << x << std::endl;
}

// 函数模板
template <typename T>
void print(T x) {
    std::cout << "模板函数：" << x << std::endl;
}

// 特化模板（更特化）
template <>
void print<double>(double x) {
    std::cout << "double特化模板：" << x << std::endl;
}

print(10); // 匹配普通函数
print(3.14); // 匹配double特化模板
print("hello"); // 匹配通用模板（T=const char*）
```

### 2.2 类模板

#### 2.2.1 基本定义与实例化

类模板的定义以`template`开头，模板参数列表后紧跟类定义，实例化类模板时必须显式指定类型参数（除非有默认模板参数）。

```cpp
// 类模板：通用的栈结构
template <typename T, size_t MAX_SIZE = 100> // 默认模板参数
class Stack {
public:
    Stack() : top_idx(0) {}
    
    void push(const T& val) {
        if (top_idx < MAX_SIZE) {
            data[top_idx++] = val;
        }
    }
    
    T pop() {
        if (top_idx > 0) {
            return data[--top_idx];
        }
        throw std::out_of_range("Stack is empty");
    }
    
private:
    T data[MAX_SIZE];
    size_t top_idx;
};

// 实例化类模板：显式指定类型，使用默认MAX_SIZE=100
Stack<int> int_stack;
int_stack.push(10);
int_stack.push(20);
std::cout << int_stack.pop() << std::endl; // 输出20

// 显式指定类型和非类型参数
Stack<double, 50> double_stack;
double_stack.push(3.14);
```

#### 2.2.2 类模板的成员函数

- 类模板的成员函数可以在类内定义（自动成为模板函数），也可在类外定义（需显式指定模板参数）；
- 成员函数只有在被调用时才会实例化。

```cpp
// 类内定义成员函数（如上Stack的push/pop）

// 类外定义成员函数示例
template <typename T, size_t MAX_SIZE>
void Stack<T, MAX_SIZE>::push(const T& val) {
    if (top_idx < MAX_SIZE) {
        data[top_idx++] = val;
    } else {
        throw std::overflow_error("Stack is full");
    }
}
```

#### 2.2.3 类模板的静态成员

类模板的静态成员属于**模板的实例化版本**，每个实例化的类有独立的静态成员，而非共享。

```cpp
template <typename T>
class MyClass {
public:
    static int count;
    MyClass() { ++count; }
};

// 静态成员的类外定义
template <typename T>
int MyClass<T>::count = 0;

// 不同实例化版本的静态成员独立
MyClass<int> a, b;
MyClass<double> c;
std::cout << MyClass<int>::count << std::endl; // 输出2
std::cout << MyClass<double>::count << std::endl; // 输出1
```

#### 2.2.4 类模板的特化

当通用模板对某些类型不适用时，可定制特化版本，分为**全特化**（所有模板参数都指定）和**偏特化**（部分模板参数指定，仅适用于类模板）。

##### 2.2.4.1 全特化

```cpp
// 通用模板
template <typename T>
class MyTemplate {
public:
    void show() {
        std::cout << "通用模板：" << typeid(T).name() << std::endl;
    }
};

// 对int类型全特化
template <>
class MyTemplate<int> {
public:
    void show() {
        std::cout << "int类型特化模板" << std::endl;
    }
};

MyTemplate<double> d;
d.show(); // 输出：通用模板：double

MyTemplate<int> i;
i.show(); // 输出：int类型特化模板
```

##### 2.2.4.2 偏特化

```cpp
// 通用模板：两个类型参数
template <typename T, typename U>
class Pair {
public:
    Pair(T a, U b) : first(a), second(b) {}
    void print() {
        std::cout << first << ", " << second << std::endl;
    }
private:
    T first;
    U second;
};

// 偏特化：第二个参数为int
template <typename T>
class Pair<T, int> {
public:
    Pair(T a, int b) : first(a), second(b) {}
    void print() {
        std::cout << "偏特化（第二个参数int）：" << first << ", " << second << std::endl;
    }
private:
    T first;
    int second;
};

Pair<double, std::string> p1(3.14, "hello");
p1.print(); // 通用模板：3.14, hello

Pair<std::string, int> p2("world", 100);
p2.print(); // 偏特化：world, 100
```

### 2.3 模板参数

#### 2.3.1 类型参数

用`typename`或`class`声明，代表任意类型（内置类型、自定义类型、模板类型），注意：

- 在模板参数列表中，`typename`和`class`无区别；
- 在嵌套依赖类型前必须用`typename`（如`typename T::iterator`）。

```cpp
template <typename T>
typename T::iterator find_first(T& container) { // 嵌套依赖类型，必须加typename
    return container.begin();
}

std::vector<int> vec = {1,2,3};
auto it = find_first(vec); // T=std::vector<int>，it=vec.begin()
```

#### 2.3.2 非类型参数

必须是**编译期可确定的常量表达式**，类型限制：

- 整数类型（int、size_t、bool 等）；
- 指针 / 引用（指向静态对象、函数、模板参数）；
- 枚举类型。

```cpp
// 非类型参数：指针（指向函数）
template <void (*Func)(int)>
void call_func(int x) {
    Func(x);
}

void print_int(int x) {
    std::cout << x << std::endl;
}

call_func<print_int>(10); // 传入函数指针，输出10

// 非类型参数：引用（静态变量）
static int val = 5;
template <int& Ref>
void modify_ref() {
    Ref += 10;
}

modify_ref<val>();
std::cout << val << std::endl; // 输出15
```

#### 2.3.3 默认模板参数

函数模板和类模板均可指定默认模板参数，C++11 后函数模板的默认参数无需显式指定空模板参数列表。

```cpp
// 函数模板默认参数
template <typename T = int, typename U = double>
auto multiply(T a, U b) {
    return a * b;
}

auto res1 = multiply(2, 3.5); // T=int，U=double，res1=7.0
auto res2 = multiply<>(); // 使用默认参数，T=int，U=double，a=0，b=0，res2=0.0

// 类模板默认参数（前文Stack示例已展示）
```

### 2.4 可变参数模板

C++11 引入可变参数模板，允许模板接受任意数量的参数（类型参数 / 非类型参数），用`...`表示参数包，通过递归展开参数包。

#### 2.4.1 基本定义与展开

```cpp
// 递归终止函数（无参数包）
void print_args() {
    std::cout << std::endl;
}

// 可变参数模板函数
template <typename T, typename... Args>
void print_args(T first, Args... rest) { // Args是参数包，rest是参数包实例
    std::cout << first << " ";
    print_args(rest...); // 递归展开参数包
}

print_args(1, 3.14, "hello", true); // 输出：1 3.14 hello true 
```

#### 2.4.2 折叠表达式（C++17）

简化参数包展开，支持`+、-、*、/、&&、||`等运算符，分为左折叠和右折叠。

```cpp
// 左折叠：( (... op arg1) op arg2 ) op arg3 ...
template <typename... Args>
auto sum_left(Args... args) {
    return (... + args); // 等价于((arg1 + arg2) + arg3) + ...
}

// 右折叠：arg1 op (arg2 op (... op argN) )
template <typename... Args>
auto sum_right(Args... args) {
    return (args + ...); // 等价于arg1 + (arg2 + (... + argN))
}

std::cout << sum_left(1,2,3,4) << std::endl; // 输出10
std::cout << sum_right(1,2,3,4) << std::endl; // 输出10

// 带初始值的折叠
template <typename... Args>
auto sum_with_init(Args... args) {
    return (0 + ... + args); // 初始值0，避免空参数包报错
}

std::cout << sum_with_init() << std::endl; // 输出0
```

#### 2.4.3 可变参数模板的应用：std::tuple

标准库的`std::tuple`是可变参数模板的典型应用，可存储任意数量、任意类型的元素。

```cpp
#include <tuple>
#include <string>

std::tuple<int, double, std::string> t(10, 3.14, "tuple demo");

// 访问tuple元素：std::get<索引>(tuple)
std::cout << std::get<0>(t) << std::endl; // 输出10
std::cout << std::get<1>(t) << std::endl; // 输出3.14
std::cout << std::get<2>(t) << std::endl; // 输出tuple demo

// 获取tuple大小：std::tuple_size
constexpr size_t size = std::tuple_size<decltype(t)>::value;
std::cout << "tuple大小：" << size << std::endl; // 输出3
```

### 2.5 模板实参的推断与转发

#### 2.5.1 完美转发（std::forward）

用于保留实参的类型（左值 / 右值），结合可变参数模板实现参数的完美转发，常用于函数包装器、工厂函数。

```cpp
// 接受任意参数并转发给其他函数
template <typename Func, typename... Args>
auto wrapper(Func&& func, Args&&... args) {
    // std::forward保留args的左值/右值属性
    return std::forward<Func>(func)(std::forward<Args>(args)...);
}

void func(int& x) { // 左值引用参数
    x += 10;
}

void func2(int&& x) { // 右值引用参数
    std::cout << "右值：" << x << std::endl;
}

int a = 5;
wrapper(func, a); // a是左值，转发给func(int&)，a变为15
wrapper(func2, 100); // 100是右值，转发给func2(int&&)，输出100
```

#### 2.5.2 类型推断与 auto

模板实参推断规则同样适用于`auto`，`auto`可看作 “隐式的模板参数”，结合`decltype`可推导表达式类型。

```cpp
template <typename T, typename U>
auto add(T a, U b) -> decltype(a + b) {
    return a + b;
}

// auto等价于模板推导
auto x = add(1, 3.14); // x的类型为double，等价于decltype(1+3.14)
```

### 2.6 模板的编译与实例化

#### 2.6.1 编译阶段

模板编译分为两个阶段：

1. 模板定义阶段：检查语法错误（如关键字拼写、分号缺失），不检查类型相关错误；
2. 模板实例化阶段：根据实参类型检查类型相关错误（如成员访问、运算符重载）。

#### 2.6.2 实例化位置

- 隐式实例化：当使用模板（如调用模板函数、创建类模板对象）时，编译器自动实例化；
- 显式实例化：通过`template`关键字强制实例化，避免多次实例化提升编译效率。

```cpp
// 显式实例化函数模板
template void swap<int>(int&, int&);

// 显式实例化类模板
template class Stack<double>;
```

## 三、小结

模板是 C++ 泛型编程的核心，通过编写与类型无关的代码实现高效复用，核心要点：

1. 函数模板：自动推导类型参数，支持重载和默认参数，推导规则忽略顶层 const；
2. 类模板：需显式指定类型参数实例化，支持静态成员、特化（全特化 / 偏特化）；
3. 模板参数：分为类型参数（typename/class）和非类型参数（编译期常量），支持默认值；
4. 可变参数模板：接受任意数量参数，通过递归或折叠表达式展开，适配参数个数不确定的场景；
5. 完美转发：结合`std::forward`保留参数的左值 / 右值属性，实现参数的无损转发；
6. 编译特性：模板分两阶段编译，实例化时检查类型错误，显式实例化可优化编译效率。

模板大幅提升了 C++ 代码的灵活性和复用性，是标准库（如 STL）的实现基础，掌握模板的核心用法（特化、可变参数、转发）是编写高效泛型代码的关键。
