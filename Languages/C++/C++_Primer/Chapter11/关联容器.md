# 关联容器

***---- Li Jinsheng***

## 一、概述

本篇为《C++ Primer》（第 5 版）第 11 章读书笔记，主要内容为关联容器相关基本概述、特性、使用方法及核心原理。关联容器的核心思想是**键值对（key-value）** 存储，支持高效的基于键的查找和访问，是 C++ 标准库中实现数据组织与快速检索的核心工具。

## 二、内容

关联容器与顺序容器（如 vector、list）的核心区别：

- 顺序容器按元素的插入顺序组织和访问；
- 关联容器按**键**的大小或自定义规则排序，通过键快速查找值，底层通常基于红黑树（有序容器）或哈希表（无序容器）实现。

C++ 标准库提供的关联容器分类：

| 容器类型             | 特点                                                         |
| :------------------- | :----------------------------------------------------------- |
| `map`                | 有序键值对，键唯一，键不可修改，值可修改，默认按键升序排列   |
| `set`                | 有序集合，仅存储键，键唯一且不可修改，默认按键升序排列       |
| `multimap`           | 有序键值对，键可重复，键不可修改，值可修改                   |
| `multiset`           | 有序集合，仅存储键，键可重复且不可修改                       |
| `unordered_map`      | 无序键值对，键唯一，基于哈希表，通过哈希函数快速查找，不保证顺序 |
| `unordered_set`      | 无序集合，仅存储键，键唯一，基于哈希表                       |
| `unordered_multimap` | 无序键值对，键可重复，基于哈希表                             |
| `unordered_multiset` | 无序集合，键可重复，基于哈希表                               |

### 2.1 有序关联容器（map/set/multimap/multiset）

#### 2.1.1 初始化与赋值

有序容器的键类型必须满足**可比较**（默认使用`<`运算符，也可自定义比较规则）。

```cpp
// map初始化：键-值对用{}包裹，多个键值对用逗号分隔
#include <map>
#include <string>
std::map<std::string, int> word_count;                // 空map
std::map<std::string, int> word_count2 = {{"apple", 1}, {"banana", 2}}; // 列表初始化

// set初始化：仅存储键
#include <set>
std::set<int> num_set = {1, 2, 3, 4};                // 有序集合，自动去重
std::multiset<int> num_multiset = {1, 2, 2, 3};      // 允许重复键
```

#### 2.1.2 元素访问与遍历

- `map`的元素访问：

  - `[]`运算符：若键不存在，则插入该键并初始化值（仅适用于非 const 的 map）；
  - `at()`：若键不存在，抛出`out_of_range`异常（更安全）；
  - 迭代器访问：解引用迭代器得到`pair<const K, V>`，`first`为键，`second`为值。


```cpp
std::map<std::string, int> word_count = {{"apple", 1}, {"banana", 2}};
word_count["apple"] = 3;          // 修改已有键的值
word_count["orange"] = 4;         // 插入新键值对
int cnt = word_count.at("banana");// 安全访问，值为2

// 遍历map
for (const auto& pair : word_count) {
    // pair.first是const string（键不可修改），pair.second是int
    std::cout << pair.first << ": " << pair.second << std::endl;
}

// 遍历set
std::set<int> num_set = {3, 1, 2};
for (int num : num_set) {
    std::cout << num << " ";      // 输出：1 2 3（自动排序）
}
```

#### 2.1.3 插入与删除

- 插入操作：

  - `insert()`：插入单个键值对 / 迭代器范围 / 初始化列表，返回值因容器类型不同而异；

    - 对于`map/set`（键唯一）：返回`pair<iterator, bool>`，`bool`表示是否插入成功；
    - 对于`multimap/multiset`（键可重复）：返回指向新插入元素的迭代器。

- 删除操作：

  - `erase()`：可按键删除（返回删除的元素个数）、按迭代器删除（返回下一个迭代器）、按迭代器范围删除。


```cpp
std::map<std::string, int> word_count;
// 插入单个键值对
auto ret = word_count.insert({"apple", 1});
if (!ret.second) { // 若apple已存在，ret.second为false
    ret.first->second++; // 修改已有值
}

// 插入初始化列表
word_count.insert({{"banana", 2}, {"orange", 3}});

// 删除元素
size_t del_num = word_count.erase("apple"); // 删除键为apple的元素，返回1
auto it = word_count.erase(word_count.find("banana")); // 按迭代器删除，it指向orange

// multimap插入（允许重复键）
std::multimap<std::string, int> mm = {{"a", 1}, {"a", 2}};
mm.insert({"a", 3}); // 成功插入，mm中有3个"a"
```

#### 2.1.4 查找与计数

- `find(key)`：返回指向第一个匹配键的迭代器，若不存在则返回`end()`；
- `count(key)`：返回键出现的次数（`map/set`中仅 0 或 1，`multimap/multiset`中可大于 1）；
- `lower_bound(key)`：返回第一个不小于 key 的迭代器；
- `upper_bound(key)`：返回第一个大于 key 的迭代器；
- `equal_range(key)`：返回`pair<iterator, iterator>`，表示键等于 key 的元素范围（适用于多键容器）。

```cpp
std::multimap<std::string, int> mm = {{"a", 1}, {"a", 2}, {"b", 3}};
// 查找所有键为"a"的元素
auto range = mm.equal_range("a");
for (auto it = range.first; it != range.second; ++it) {
    std::cout << it->second << " "; // 输出：1 2
}

// 计数
size_t a_count = mm.count("a"); // a_count = 2
size_t c_count = mm.count("c"); // c_count = 0
```

#### 2.1.5 自定义比较规则

有序容器默认使用`std::less<K>`（即`<`运算符）排序，可自定义比较函数 / 仿函数。

```cpp
// 自定义仿函数：按字符串长度降序排列
struct StrLenCmp {
    bool operator()(const std::string& s1, const std::string& s2) const {
        return s1.size() > s2.size();
    }
};

// 使用自定义比较规则的map
std::map<std::string, int, StrLenCmp> m = {{"apple", 1}, {"pear", 2}, {"banana", 3}};
// 遍历结果：banana(6)、apple(5)、pear(4)（按长度降序）
for (const auto& pair : m) {
    std::cout << pair.first << ": " << pair.second << std::endl;
}
```

### 2.2 无序关联容器（unordered 系列）

#### 2.2.1 核心特性

- 底层基于**哈希表**，通过哈希函数将键映射到桶（bucket），查找、插入、删除平均时间复杂度为 O (1)；
- 不保证元素顺序，遍历顺序可能随插入 / 删除变化；
- 键类型必须满足：可哈希（默认使用`std::hash<K>`）且支持`==`运算符；
- 若键为自定义类型，需自定义哈希函数和`==`运算符。

#### 2.2.2 基本使用

```cpp
#include <unordered_map>
#include <unordered_set>

// unordered_map基本操作
std::unordered_map<std::string, int> um = {{"apple", 1}, {"banana", 2}};
um["orange"] = 3; // 插入新元素，位置由哈希函数决定

// 遍历：顺序不固定
for (const auto& pair : um) {
    std::cout << pair.first << ": " << pair.second << std::endl;
}

// unordered_set去重
std::unordered_set<int> us = {3, 1, 2, 2};
for (int num : us) {
    std::cout << num << " "; // 输出：1 2 3（顺序不固定，去重）
}
```

#### 2.2.3 哈希冲突与桶操作

哈希冲突：不同的键哈希到同一个桶，会降低容器性能。可通过桶操作查看 / 调整哈希表状态：

- `bucket_count()`：返回当前桶的数量；
- `max_bucket_count()`：返回最大允许的桶数量；
- `bucket_size(n)`：返回第 n 个桶中的元素个数；
- `bucket(key)`：返回键对应的桶编号；
- `rehash(n)`：重新分配桶，使桶数量至少为 n；
- `reserve(n)`：预留空间，使容器可存储 n 个元素而无需重新哈希。

```cpp
std::unordered_map<std::string, int> um = {{"a",1},{"b",2},{"c",3}};
std::cout << "桶数量：" << um.bucket_count() << std::endl;
std::cout << "键a的桶编号：" << um.bucket("a") << std::endl;
std::cout << "桶0的元素数：" << um.bucket_size(0) << std::endl;

// 预分配桶，减少哈希冲突
um.rehash(100); // 桶数量至少为100
```

#### 2.2.4 自定义类型作为键

自定义类型作为无序容器的键，需满足两个条件：

1. 重载`==`运算符，用于判断键是否相等；
2. 自定义哈希函数（或特化`std::hash`模板）。

```cpp
#include <functional>

// 自定义类型：坐标
struct Point {
    int x, y;
    // 重载==运算符
    bool operator==(const Point& other) const {
        return x == other.x && y == other.y;
    }
};

// 自定义哈希函数
struct PointHash {
    size_t operator()(const Point& p) const {
        // 组合x和y的哈希值，避免冲突
        return std::hash<int>()(p.x) ^ (std::hash<int>()(p.y) << 1);
    }
};

// 使用自定义类型的unordered_set
std::unordered_set<Point, PointHash> us;
us.insert({1, 2});
us.insert({3, 4});

// 查找
if (us.find({1, 2}) != us.end()) {
    std::cout << "找到点(1,2)" << std::endl;
}
```

### 2.3 关联容器的选择与性能

- 有序容器（map/set）：

  - 优势：有序遍历、支持范围查找（如`lower_bound`/`upper_bound`）；
  - 劣势：插入 / 查找时间复杂度 O (log n)，依赖红黑树的平衡操作。

  

- 无序容器（unordered 系列）：

  - 优势：平均 O (1) 的插入 / 查找效率，适合高频查找场景；
  - 劣势：哈希冲突会导致性能下降，不保证顺序，自定义键需额外处理哈希。

  

**选择原则**：

1. 若需要有序遍历或范围查找 → 选有序容器；
2. 若仅需快速查找 / 插入，且不关心顺序 → 选无序容器；
3. 若键可重复 → 选 multimap/multiset 或 unordered_multimap/unordered_multiset。

### 2.4 关联容器的迭代器

- 关联容器的迭代器均为**双向迭代器**（不支持随机访问，如`it += 1`）；
- `map/multimap`的迭代器解引用得到`pair<const K, V>`，键不可修改（`const`），值可修改（除非容器是 const）；
- `set/multiset`的迭代器解引用得到`const K`，键完全不可修改；
- 无序容器的迭代器特性与有序容器一致，仅顺序不同。

```cpp
std::map<std::string, int> m = {{"apple", 1}};
auto it = m.begin();
// it->first = "orange"; // 错误：键是const，不可修改
it->second = 2; // 正确：值可修改

std::set<int> s = {1};
auto it2 = s.begin();
// *it2 = 2; // 错误：set的元素是const
```

## 三、小结

关联容器是 C++ 标准库中高效组织键值数据的核心工具，核心分为有序（红黑树实现）和无序（哈希表实现）两大类：

1. 有序容器（map/set/multimap/multiset）：按键排序，支持范围查找，适合需要有序遍历的场景；
2. 无序容器（unordered 系列）：哈希表实现，平均 O (1) 查找效率，适合高频查找、不关心顺序的场景；
3. 键的特性决定容器选择：唯一键选 map/set，重复键选 multimap/multiset，自定义键需满足比较（有序）或哈希 / 相等（无序）规则；
4. 迭代器特性：关联容器迭代器为双向迭代器，map 的键、set 的元素均不可修改。

合理选择关联容器并掌握其核心操作（插入、删除、查找），是提升 C++ 程序数据检索效率的关键。
