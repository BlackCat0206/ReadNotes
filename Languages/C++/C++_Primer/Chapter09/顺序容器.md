## 一、概述

本篇为《C++ Primer》（第 5 版）第 9 章读书笔记，主要内容为 C++ 标准库中的顺序容器相关基本概述、特性与使用方式。顺序容器（sequential container）将元素按顺序组织，元素的访问位置与其加入容器的位置相关，是 C++ 程序中存储和管理一组元素的核心工具。

## 二、内容

顺序容器的基本思想：**有序存储与灵活访问**。以线性序列的方式存储元素，支持按位置访问、插入、删除等操作，不同容器针对不同操作场景做了性能优化，满足不同的业务需求。

### 2.1 顺序容器类型

C++ 标准库提供的顺序容器主要包括：

- `vector`：可变大小数组，支持快速随机访问，尾部插入 / 删除效率高，中间插入 / 删除效率低；
- `string`：专门存储字符的容器，与`vector<char>`类似，提供额外的字符串操作接口；
- `deque`：双端队列，支持快速随机访问，头尾插入 / 删除效率高；
- `list`：双向链表，只支持双向顺序访问，任意位置插入 / 删除效率高，随机访问效率低；
- `forward_list`：单向链表，只支持单向顺序访问，开销比`list`更小，C++11 新增；
- `array`：固定大小数组，支持快速随机访问，不能动态扩容，C++11 新增。

各容器核心特性对比：

| 容器类型       | 随机访问     | 尾部插入 / 删除 | 头部插入 / 删除 | 中间插入 / 删除 | 动态扩容 |
| :------------- | :----------- | :-------------- | :-------------- | :-------------- | :------- |
| `vector`       | 支持（快）   | 快              | 慢              | 慢              | 支持     |
| `string`       | 支持（快）   | 快              | 慢              | 慢              | 支持     |
| `deque`        | 支持（快）   | 快              | 快              | 慢              | 支持     |
| `list`         | 不支持（慢） | 快              | 快              | 快              | 支持     |
| `forward_list` | 不支持（慢） | 快              | 快              | 快              | 支持     |
| `array`        | 支持（快）   | 不支持          | 不支持          | 不支持          | 不支持   |

### 2.2 容器迭代器

所有顺序容器都支持迭代器（iterator），用于遍历和访问容器元素，迭代器的基本操作：

- `*iter`：解引用，返回迭代器指向的元素；
- `iter->mem`：等价于`(*iter).mem`，访问元素的成员；
- `++iter`/`--iter`：移动迭代器（`forward_list`仅支持`++iter`）；
- `iter1 == iter2`/`iter1 != iter2`：判断迭代器是否指向同一位置。

#### 2.2.1 迭代器范围

迭代器范围遵循 “左闭右开” 原则：`[begin, end)`，`begin`指向第一个元素，`end`指向最后一个元素的下一个位置（尾后迭代器），尾后迭代器不可解引用。

```cpp
#include <vector>
#include <iostream>
using namespace std;

int main() {
    vector<int> vec = {1, 2, 3, 4, 5};
    // 遍历容器
    for (auto iter = vec.begin(); iter != vec.end(); ++iter) {
        cout << *iter << " "; // 输出：1 2 3 4 5
    }
    return 0;
}
```

#### 2.2.2 迭代器失效

容器操作可能导致迭代器失效（指向非法位置），需特别注意：

- `vector`/`string`：扩容时所有迭代器失效；尾部插入 / 删除仅尾后迭代器失效；中间插入 / 删除，插入 / 删除位置后的迭代器失效；
- `deque`：头尾插入仅尾后迭代器失效；中间插入 / 删除所有迭代器失效；
- `list`/`forward_list`：插入 / 删除操作不会导致迭代器失效（指向被删除元素的迭代器除外）；
- `array`：迭代器始终有效（除非数组本身被销毁）。

### 2.3 容器的构造与初始化

#### 2.3.1 通用构造方式

```cpp
#include <vector>
#include <list>
#include <array>
#include <string>
using namespace std;

int main() {
    // 1. 默认构造：空容器（array需指定大小）
    vector<int> vec1;
    list<string> lst1;
    array<int, 5> arr1; // 固定大小为5，元素默认初始化

    // 2. 拷贝构造：拷贝另一个容器的所有元素（类型需匹配）
    vector<int> vec2(vec1);
    list<string> lst2(lst1);

    // 3. 范围构造：拷贝迭代器范围的元素（类型可转换）
    vector<int> vec3 = {1, 2, 3};
    list<int> lst3(vec3.begin(), vec3.end());

    // 4. 数值初始化：指定元素个数和初始值
    vector<int> vec4(10, 0); // 10个元素，每个值为0
    string str(5, 'a');     // 5个'a'，结果为"aaaaa"

    // 5. 列表初始化（C++11）
    vector<int> vec5 = {1, 2, 3, 4, 5};
    array<int, 5> arr2 = {1, 2, 3, 4, 5}; // array必须指定大小，且初始化列表元素数不超过大小
    return 0;
}
```

#### 2.3.2 特殊说明

- `array`：初始化时必须指定大小，且大小是编译期常量；默认构造的`array`元素是默认初始化（基本类型为未定义值）；
- `string`：额外支持 C 风格字符串初始化（`string str = "hello"`）；
- `forward_list`：不支持`size()`成员函数（需手动遍历统计）。

### 2.4 容器的基本操作

#### 2.4.1 大小与容量操作

```cpp
#include <vector>
#include <iostream>
using namespace std;

int main() {
    vector<int> vec = {1, 2, 3};
    // 大小：当前元素个数
    cout << "size: " << vec.size() << endl; // 输出3
    // 判空：是否无元素
    cout << "empty: " << boolalpha << vec.empty() << endl; // 输出false
    // 最大容量：容器可容纳的最大元素数（与系统相关）
    cout << "max_size: " << vec.max_size() << endl;

    // vector/string 特有：容量（已分配空间可容纳的元素数）
    cout << "capacity: " << vec.capacity() << endl;
    // 预留空间：避免频繁扩容
    vec.reserve(100); // 预留至少100个元素的空间，不改变size
    // 收缩容量：匹配当前size（C++11）
    vec.shrink_to_fit();
    return 0;
}
```

#### 2.4.2 元素访问操作

```cpp
#include <vector>
#include <list>
#include <deque>
#include <iostream>
using namespace std;

int main() {
    vector<int> vec = {1, 2, 3, 4, 5};
    deque<int> deq = {10, 20, 30};
    list<int> lst = {100, 200, 300};

    // 1. 下标访问：仅支持vector/string/deque/array，不检查越界
    cout << vec[0] << endl;   // 输出1
    cout << deq[1] << endl;   // 输出20

    // 2. at()访问：支持vector/string/deque/array，越界抛异常
    cout << vec.at(2) << endl; // 输出3

    // 3. 首尾元素访问：所有顺序容器支持
    cout << vec.front() << endl; // 输出1
    cout << vec.back() << endl;  // 输出5
    // list/forward_list 无下标，只能通过迭代器访问
    auto iter = lst.begin();
    cout << *iter << endl; // 输出100

    return 0;
}
```

#### 2.4.3 插入与删除操作

##### 通用插入（`insert`）

```cpp
#include <vector>
#include <string>
#include <iostream>
using namespace std;

int main() {
    vector<string> vec = {"apple", "banana"};
    // 1. 单个元素插入：在指定位置前插入
    vec.insert(vec.begin() + 1, "orange"); // vec: ["apple", "orange", "banana"]

    // 2. 多个元素插入：插入n个相同元素
    vec.insert(vec.end(), 2, "grape"); // vec: ["apple", "orange", "banana", "grape", "grape"]

    // 3. 范围插入：插入迭代器范围的元素
    vector<string> tmp = {"pear", "mango"};
    vec.insert(vec.begin(), tmp.begin(), tmp.end()); // vec: ["pear", "mango", "apple", "orange", "banana", "grape", "grape"]

    // 4. 列表插入（C++11）
    vec.insert(vec.end(), {"watermelon", "cherry"});
    return 0;
}
```

##### 通用删除（`erase`）

```cpp
#include <vector>
#include <iostream>
using namespace std;

int main() {
    vector<int> vec = {1, 2, 3, 4, 5, 6};
    // 1. 删除单个元素
    vec.erase(vec.begin() + 2); // 删除3，vec: [1,2,4,5,6]

    // 2. 删除范围元素（左闭右开）
    vec.erase(vec.begin() + 1, vec.begin() + 3); // 删除2、4，vec: [1,5,6]

    // 3. 清空容器（等价于erase(vec.begin(), vec.end())）
    vec.clear();
    return 0;
}
```

##### 特殊容器的插入 / 删除

- `list`/`forward_list`：提供`push_front`/`pop_front`（`vector`/`string`不支持），且插入 / 删除效率更高；
- `forward_list`：无`insert`/`erase`，而是`insert_after`/`erase_after`（操作指定位置的下一个元素）；
- `array`：不支持插入 / 删除（大小固定）。

### 2.5 容器适配器

容器适配器（container adapter）是对顺序容器的封装，提供特定的操作接口，包括`stack`（栈）、`queue`（队列）、`priority_queue`（优先队列）。

#### 2.5.1 适配器的基本特性

- 适配器不直接存储元素，而是依赖底层容器（可指定，默认：`stack`/`queue`默认`deque`，`priority_queue`默认`vector`）；
- 适配器不支持迭代器，只能通过自身接口访问元素。

#### 2.5.2 栈（`stack`）

```cpp
#include <stack>
#include <iostream>
using namespace std;

int main() {
    stack<int> stk;
    // 入栈
    stk.push(1);
    stk.push(2);
    stk.push(3);

    // 访问栈顶元素
    cout << "栈顶：" << stk.top() << endl; // 输出3

    // 出栈
    stk.pop(); // 删除栈顶元素（3）
    cout << "栈顶：" << stk.top() << endl; // 输出2

    // 大小与判空
    cout << "大小：" << stk.size() << endl; // 输出2
    cout << "空：" << boolalpha << stk.empty() << endl; // 输出false
    return 0;
}
```

#### 2.5.3 队列（`queue`）与优先队列（`priority_queue`）

```cpp
#include <queue>
#include <iostream>
using namespace std;

int main() {
    // 普通队列：先进先出
    queue<int> q;
    q.push(1);
    q.push(2);
    q.push(3);
    cout << "队首：" << q.front() << endl; // 输出1
    cout << "队尾：" << q.back() << endl;  // 输出3
    q.pop(); // 删除队首元素（1）
    cout << "队首：" << q.front() << endl; // 输出2

    // 优先队列：默认最大值优先
    priority_queue<int> pq;
    pq.push(5);
    pq.push(2);
    pq.push(8);
    cout << "堆顶：" << pq.top() << endl; // 输出8
    pq.pop(); // 删除堆顶元素（8）
    cout << "堆顶：" << pq.top() << endl; // 输出5
    return 0;
}
```

### 2.6 顺序容器的选择原则

选择容器需结合操作场景的性能需求：

1. 需随机访问且尾部操作频繁：优先`vector`；
2. 需字符串专属操作：`string`；
3. 需头尾操作频繁且随机访问：`deque`；
4. 需任意位置插入 / 删除频繁：`list`（双向遍历）或`forward_list`（单向遍历，开销更小）；
5. 大小固定且需高效随机访问：`array`；
6. 需栈 / 队列 / 优先队列语义：对应适配器（`stack`/`queue`/`priority_queue`）。

### 2.7 容器的算法使用

顺序容器可结合标准库算法（`<algorithm>`）进行操作，如排序、查找、遍历等：

```cpp
#include <vector>
#include <algorithm>
#include <iostream>
using namespace std;

int main() {
    vector<int> vec = {3, 1, 4, 1, 5, 9};
    // 排序
    sort(vec.begin(), vec.end()); // vec: [1,1,3,4,5,9]
    // 查找
    auto iter = find(vec.begin(), vec.end(), 5);
    if (iter != vec.end()) {
        cout << "找到5，位置：" << iter - vec.begin() << endl; // 输出4
    }
    // 遍历（for_each）
    for_each(vec.begin(), vec.end(), [](int x) {
        cout << x << " "; // 输出1 1 3 4 5 9
    });
    return 0;
}
```

注：`list`/`forward_list`不支持随机访问的算法（如`sort`），需使用自身的`sort`成员函数：

```cpp
list<int> lst = {3, 1, 4, 1, 5, 9};
lst.sort(); // 链表专属排序
```

## 三、小结

C++ 顺序容器是存储和管理元素的核心工具，核心要点：

1. 不同顺序容器的性能特性不同，需根据操作场景（随机访问、插入 / 删除位置）选择合适的容器；
2. 迭代器是访问容器的关键，需注意迭代器失效问题；
3. 容器适配器封装了顺序容器，提供栈、队列等特定语义的操作；
4. 结合标准库算法可高效实现容器的排序、查找等操作，`list`/`forward_list`需使用自身成员函数替代部分算法；
5. `array`作为固定大小数组，相比原生数组更安全，`string`作为字符容器，提供丰富的字符串操作接口。

掌握顺序容器的特性和使用方式，能高效实现元素的存储、访问和修改，提升程序的可读性和性能。
