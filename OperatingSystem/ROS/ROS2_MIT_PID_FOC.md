### 一、MIT 模式的核心原理

MIT 模式的本质是 “位置环 + 速度环 + 前馈力矩” 的并联控制结构，通过整合多维度参数计算参考力矩，再转换为电机驱动电流，其核心计算公式为：

Tref=kp×(pdes−θm)+kd×(vdes−dθm)+tff

各参数含义：

- Tref：电机参考力矩（单位：N⋅m），是控制电机运动的核心指令；
- kp：位置增益（比例系数），用于减小电机实际位置与期望位置的偏差；
- kd：速度增益（微分系数），用于抑制电机运动中的震荡，提升稳定性；
- pdes：电机输出轴的期望位置（单位：rad），由上层运动规划或手柄控制指令生成；
- θm：电机输出轴的当前位置（单位：rad），通过编码器实时反馈；
- vdes：电机输出轴的期望速度（单位：rad/s），配合位置指令实现平滑运动；
- dθm：电机输出轴的当前速度（单位：rad/s），由编码器数据实时计算；
- tff：前馈力矩，用于补偿重力、摩擦力等干扰，提前抵消负载影响，提升响应速度。

### 二、MIT 模式的控制流程

1. **参数输入与反馈采集**
   - 输入参数：由上层控制（如手柄操作、步态规划算法）提供期望位置pdes、期望速度vdes，并预设位置增益kp、速度增益kd及前馈力矩tff；
   - 反馈数据：通过电机编码器实时采集当前位置θm、当前速度dθm，形成闭环控制的反馈依据。
2. **参考力矩计算**
   - 位置环计算：通过kp×(pdes−θm)计算位置偏差补偿力矩，偏差越大，补偿力矩越强，快速拉近距离期望位置；
   - 速度环计算：通过kd×(vdes−dθm)计算速度偏差补偿力矩，抑制电机加速过快或震荡，保障运动平稳；
   - 前馈叠加：加入tff补偿固定干扰（如关节重力负载），减少闭环控制的调节压力，提升动态响应。
3. **电流转换与电机驱动**
   - 力矩 - 电流换算：参考力矩Tref通过电机扭矩常数KT换算为参考电流iqref（iqref=Tref/KT），同时配合直轴电流idref（通常设为 0，优化转矩输出）；
   - FOC 驱动：基于磁场定向控制方法，将iqref和idref转换为三相交流电，精准控制电机定子磁场方向与强度，实现力矩的高效输出。

### 三、MIT 模式的关键应用场景

根据参数配置的不同，MIT 模式可适配多种运动需求，核心应用场景包括：

1. **匀速转动控制（\**kp=0,kd\\=0\**）**
   - 仅通过速度环控制，给定vdes即可实现电机匀速转动；
   - 注意：kd不宜过大，否则会导致电机启动震荡，需根据电机特性微调。
2. **恒定力矩输出（\**kp=0,kd=0\**）**
   - 仅通过前馈力矩tff控制，电机持续输出恒定力矩；
   - 限制：若电机空转或负载过小，即使tff较大，电机也会持续加速至最大转速，无法达到目标力矩（需配合负载反馈优化）。
3. **定点控制（\**kp\\=0,kd\\=0\**，\**pdes\**为常量，\**vdes=0\**）**
   - 位置环主导，速度环抑制震荡，电机逐步趋近并稳定在pdes对应的位置，适用于机器人关节的精准定位（如站立姿态保持）。
4. **轨迹跟踪（\**kp\\=0,kd\\=0\**，\**pdes\**随时间变化，\**vdes\**为\**pdes\**的导数）**
   - 位置环与速度环协同，电机按预设轨迹（如步态规划的关节运动曲线）运动，同时通过速度反馈避免轨迹偏移，适用于行走、摆腿等动态动作。

### 四、与传统 PID 控制的区别

1. 传统 PID 多为 “位置环串联速度环” 的结构，MIT 模式为 “位置环 + 速度环并联 + 前馈”，响应速度更快，抗干扰能力更强；
2. 引入前馈力矩tff，提前补偿固定负载（如关节重力），减少闭环调节的滞后，避免低速爬行或启动冲击；
3. 直接输出参考力矩，与 FOC 方法结合更紧密，适配四足机器人关节的动态负载变化（如行走时的腿部受力波动）。

### 五、工程实现中的关键细节

1. **参数整定**：kp和kd需根据电机特性（如额定力矩、响应速度）和关节负载（如腿部重量）调整，一般通过 “先调kd抑制震荡，再调kp提升响应” 的顺序优化；
2. **通信支持**：控制指令与反馈数据通过 CANFD 总线传输，保障高波特率（最高 5Mbps）和低延迟，适配电机实时控制需求；
3. **安全机制**：电机启动前需进行关节使能，停止时需失能关节，避免误操作导致的机械碰撞；同时通过电流监测防止电机过载。

# 二、工程中的示例

### 一、增益变化的核心逻辑

MIT 模式的增益调整本质是 “平衡响应速度与稳定性”：

- kp 决定位置偏差的补偿强度：kp 越大，位置跟踪越精准、响应越快，但过大易导致关节震荡；
- kd 提供阻尼效应：kd 越大，运动越平稳、抗干扰能力越强，但过大易导致启动迟滞、响应变慢；
- 不同场景对 “响应速度” 和 “稳定性” 的优先级要求不同，因此需要针对性调整 kp 和 kd 的组合。

### 二、典型场景的增益配置差异

结合实训手册中的实验场景，具体适配情况如下：

| 应用场景                   | 核心需求             | **kp** 配置特点            | **kd** 配置特点          | 示例说明                                                     |
| :------------------------- | :------------------- | :------------------------- | :----------------------- | :----------------------------------------------------------- |
| 匀速转动（如连续摆腿）     | 运动平稳、无震荡     | 设为 0（不依赖位置环）     | 中等大小（抑制加速震荡） | 手册中 “kp=0,kd=0” 模式，kd 不宜过大，避免启动时出现抖动。  |
| 恒定力矩输出（如支撑腿）   | 力矩稳定、负载抵抗强 | 设为 0                     | 设为 0                   | 仅依赖前馈力矩 tff，无需位置 / 速度反馈调节，增益无作用。    |

### 三、工程实现中的调整方式

1. **按功能模块预设增益**：在源码中为不同运动模式（如站立、行走、摆腿）预设固定增益组合，例如：
   - 站立模式：kp=180.0,kd=8.0（强位置约束，稳定姿态）；
   - 摆腿测试：kp=20.0,kd=5.0（灵活响应，避免震荡）。
2. **通过配置文件动态加载**：在 `gazebo.yaml` 等配置文件中定义各控制器的增益参数，无需修改源码即可调整，适配不同测试场景。
3. **根据负载动态适配**：对于足端触地等负载变化场景，通过传感器反馈的力矩 / 电流数据，实时微调增益（如触地时增大 kd 提升稳定性）。

简单来说，增益参数并非固定值，而是根据 “是否需要精准定位”“是否有动态负载”“是否需快速响应” 等场景需求灵活配置，核心目标是让电机控制在该场景下达到 “响应速度” 与 “稳定性” 的最优平衡。
