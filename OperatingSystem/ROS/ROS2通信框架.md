# ROS2通信框架

---- Chime Lee

****

ROS2 底层 DDS 主流的框架主要有 Fast DDS、Cyclone DDS、Connext DDS 这三款，也是官方适配的核心实现，三者在资源消耗和通信速度上的差异很明显：

1. **Fast DDS**（原 Fast-RTPS）：是 ROS2 默认的 DDS 框架，通信速度是三者里偏快的，传输低延迟、实时性不错，对传输带宽的利用效率也比较高；资源消耗属于中等水平，在嵌入式设备上适配性也挺好，兼顾了性能和资源占用，也是我做四足机器人 ROS2 开发时主要用的。
2. **Cyclone DDS**：资源消耗是三者里最低的，对内存、CPU 的占用都比较小，特别适合无人机、四足机器人这类嵌入式硬件，轻量化的优势很明显；通信速度稍逊于 Fast DDS，不过在中小数据量的传输场景下，实时性和稳定性都能满足需求，是嵌入式 ROS2 开发里很常用的轻量化选择。
3. **Connext DDS**：商业级的 DDS 框架，通信速度和实时性是三者里最优的，对大数据量、高并发的传输支持更好，可靠性也更高；但短板是资源消耗最大，对 CPU 和内存的要求比较高，一般更适合算力充足的工业端、服务端设备，嵌入式端除非硬件算力足够，否则很少用。

还有个小众的**Iceoryx**，是基于共享内存的轻量级通信框架，不算传统 DDS，但常和 DDS 配合用在 ROS2 嵌入式场景，它的资源消耗极低，进程间通信速度极快，适合嵌入式设备内部的高频数据交互。
